# 4-SuzKnobWetter

制作期間 : 約6ヵ月

SuzKnobWetterは、DAW上で音声信号に対して空間的な深み(リバーブ成分)を直感的に付加できる非線形プラグインです。複雑なパラメータを排除し、一つのノブで「音の濡れ具合」を自在にコントロールすることに特化しており、ミキシング時の素早い意思決定をサポートします。模倣元は「[OneKnobWetter](https://wavesjapan.jp/plugins/oneknob-wetter)」というWavesの提供しているプラグインです。

## 技術的な工夫点

本アプリケーションは、プロフェッショナルな制作環境での使用に耐えうるパラメータ管理と、低遅延な信号処理を両立させました。また非線形プラグインにすることで効果のかかり過ぎを防いでいます

* **APVTSによる堅牢なパラメータ管理**
    `juce::AudioProcessorValueTreeState`を採用し、プラグインのパラメータ(Wetterノブ、ON/OFF状態)を一元管理しています。これにより、DAW側でのオートメーション記録や、プロジェクト保存時の設定再現を確実に行える設計にしました。
  
* **Listenerパターンを用いたUIと内部処理の完全同期**
    `juce::AudioProcessorValueTreeState::Listener`を継承し、内部パラメータが変更された際に parameterChanged メソッドを介してUI上のテキスト表示などを即座に書き換えるロジックを実装しました。これにより、ノブ操作と画面表示の乖離をなくしています。
  
* **JUCE純正DSPモジュールの最適化利用**
    高品質な空間表現を低負荷で実現するため、`juce::Reverb`クラスを内部エンジンに採用しました`processBlock`内でリアルタイムにDry/Wetの比率を演算し、オーディオバッファへ書き戻すことで、音質劣化を最小限に抑えた処理系を構築しています。

## ユーザー観点での工夫

「迷わず、素早く、理想の響きを作る」というユーザー体験(UX)を最優先に設計しました。

* **ワンノブ・オペレーションによる直感的な操作**
    あえて多くのパラメータを一つの「Wetter」ノブに集約しました。エンジニアやクリエイターが「もう少し空間を広げたい」と感じた瞬間、迷うことなくノブを回すだけで理想の響きに到達できるよう、パラメータのカーブや反応を調整しています。単一ノブなので初心者にもやさしい設計になっています。
  
* **視認性の高いON/OFF状態の提示**
    エフェクトの有効/無効を切り替えるボタンには、状態に応じてテキストが「ON/OFF」と切り替わる動的なフィードバックを実装しました。一目で現在のプラグインの状態が把握できるため、ミキシング中のケアレスミスを防ぎます。

* **UIの削減、プラグインの軽量化**
    楽曲制作をしているとプラグインを複数挿すような時がよくあります。その際に起きる問題として「DAWの動きが重くなる」「処理が上手く行われない」などがあります。そんな問題を解決するためこのプラグインでは視認性が悪くならない程度にUIを削減し、より実用的なプラグインとして使えるように設計しています。

* **非線形プラグイン**
    非線形プラグインとは入力値(ノブ)に対し出力値が比例しないプラグインの事であり、メリットとしては効果のかかり過ぎを防ぐことが出来ます。そのため、ノブの操作が苦手な方や楽曲制作が初心者の方でも扱いやすいプラグインになっています。

## 課題に対するアプローチ

* **スレッド間のデータ整合性の確保**
    オーディオ処理を行う「オーディオスレッド」と、GUIを管理する「メッセージスレッド」の間でデータの不整合やクラッシュが起きないよう、スレッドセーフなパラメータ取得(`getRawParameterValue`)を徹底しました。これにより、激しいオートメーション操作時でもノイズやフリーズが発生しない安定性を確保しました。

## エピソード

* **制作の流れ、きっかけ、得たこと**
    楽曲制作ではボーカルや音源に対し「wet」をかけることが多々あります。その際「[OneKnobWetter](https://wavesjapan.jp/plugins/oneknob-wetter)」のようなプラグインが使用されるわけですが、その模倣プラグインを本家より少し扱い安くして制作出来たら便利だと思い、制作しました。その際こだわったことは「非線形プラグインにすること」です。しかし、制作初期段階では非線形プラグインについての知識が浅かったため、数々の非線形音響学の文献を読みました。イメージを数値化する作業だったので、検証は耳に頼るほかありませんでした。そのため試行回数が過去最多になり、自分の納得のいく完成までに時間がかかりました。得たこととして、ユーザー視点での使いやすさを重視して制作を行うと、より完成度の高いものが出来るという事です。特に視覚的、聴覚的なことに干渉するプラグイン制作ではそれが如実に表れるため、納得のいくまで試行を重ねることの重要さを理解しました。
